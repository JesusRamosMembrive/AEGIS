## Rules of good coding (20 rules)
1. Avoid magic numbers and strings
2. Use meaningful names for variables and functions
3. Avoid deep nesting of code blocks
4. Avoid long parameter lists
5. Keep functions small and focused
6. Keep code DRY (Don't Repeat Yourself)
7. Apply KISS (Keep It Simple, Stupid) principle
8. Prefer composition over inheritance
9. Comment why, not what
10. Write unit tests for critical functionality
11. Use version control effectively
12. Refactor regularly to improve code quality
13. Follow consistent coding style and conventions
14. Handle errors and exceptions gracefully
15. Optimize only after measuring performance bottlenecks
16. Use design patterns judiciously
17. Keep dependencies up to date
18. Document public APIs clearly
19. Review code with peers regularly
20. Continuously learn and improve coding skills
